# Proyecto_02_-Tema-
Colaboración con GitHub donde se estará demostrando el proceso de la realización del Proyecto 02 del curso: Programación para Estadística II.

Desarrollo de la Aplicación Shiny en Python

1. La aplicación debe estar desarrollada utilizando Shiny para Python (documentación
oficial).
2. Debe permitir la visualización y análisis interactivo del conjunto de datos
seleccionado.
3. Requisitos técnicos mínimos:
4. Interfaz gráfica intuitiva y funcional.
5. Al menos tres tipos de visualizaciones (por ejemplo, gráficos de barras y mapas de
calor).
6. Opciones para filtrar o modificar los datos visualizados según parámetros definidos
por el usuario.
7. Código bien estructurado, comentado y siguiendo buenas prácticas de
programación.


Selección y Análisis del Conjunto de Datos
1. Criterios de selección:
2. El conjunto de datos debe ser público, relevante y suficiente para realizar un
análisis significativo.
3. Debe poseer al menos 500 registros y contener variables de distintos tipos
(numéricas, categóricas, etc.).
4. Explicación y justificación:
5. Se debe explicar por qué se eligió ese conjunto de datos: relevancia, interés
académico o social, potencial para el análisis.
6. Incluir una breve descripción del origen del conjunto de datos y sus principales
características.
7. Definición de objetivos y preguntas:
8. Presentar los objetivos concretos del análisis.
9. Formular al menos tres preguntas que vuestra aplicación ayudará a responder.


Presentación del Análisis
1. La aplicación debe incluir una sección de descripción del conjunto de datos y su
contexto.
2. Incluir visualizaciones claras y explicativas que respondan a las preguntas
planteadas.
3. Aportar comentarios y conclusiones preliminares basadas en los resultados
obtenidos.
4. La presentación debe ser autocontenida, permitiendo a cualquier usuario
comprender el propósito y los hallazgos sin consultar fuentes externas.


Colaboración con Git
1. El desarrollo debe realizarse en un repositorio Git compartido (por ejemplo, en
GitHub o GitLab).
2. Todos los integrantes deben contribuir activamente, quedando reflejado en el
historial de commits.
3. Se requiere evidencia de:
4. Creación y uso de ramas para el desarrollo de funcionalidades.
5. Realización de commits descriptivos y frecuentes.
6. Ejecución de pull y push para sincronización de cambios.
7. Realización de merge de ramas secundarias con la rama principal (main o master).
8. Incluid capturas de pantalla o enlaces al historial del repositorio como evidencia en
el informe final.


Entrega y Evaluación
1. Formato de entrega:
2. Código fuente de la aplicación en el repositorio Git.
3. Informe en PDF que incluya:
4. Descripción del grupo y roles.
5. Justificación y análisis del conjunto de datos.
6. Preguntas y objetivos planteados.
7. Capturas de la aplicación en funcionamiento.
8. Evidencia del uso de Git (capturas o enlaces).
9. Plazos:
10. La fecha límite de entrega será comunicada por el docente y respetará el formato
23/11/2025.
